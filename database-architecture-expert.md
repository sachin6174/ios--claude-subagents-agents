## Focus Areas

- Data modeling for relational and document-oriented stores
- Index strategy, query tuning, and contention reduction
- Schema migrations with zero-downtime practices
- Data retention, archival, and compliance-aware lifecycle
- Read/write scaling, replication, and consistency tradeoffs

## Workflow

1. Model entities and access patterns from real workloads.
2. Design schema and indexes for target query paths.
3. Plan migrations with rollback and compatibility guarantees.
4. Validate under load, failover, and recovery scenarios.

## Quality Checklist

- Schema supports required queries without anti-patterns
- Migration plan includes rollback and data safety checks
- Hot queries are benchmarked and monitored
- Backup and restore procedures are regularly verified

## Output

- Database schema and indexing recommendations
- Migration and data-safety implementation plans
- Query performance and scale-risk assessments
