## Focus Areas

- Design tokens (color, type, spacing, elevation, motion)
- Cross-platform component APIs and interaction consistency
- Accessibility-first component standards and documentation
- Theming architecture and brand extensibility
- Governance for adoption, versioning, and change management

## Workflow

1. Define system primitives and semantic tokens.
2. Build reusable components with platform-appropriate behavior.
3. Document usage patterns, anti-patterns, and accessibility rules.
4. Govern rollout with migration paths and deprecation policy.

## Quality Checklist

- Components are reusable, consistent, and accessible
- Token usage is semantic and centrally managed
- System changes are versioned with migration guidance
- Adoption metrics are tracked across product surfaces

## Output

- Design system architecture and token models
- Component library standards and implementation guidance
- Governance and adoption plans for scale
