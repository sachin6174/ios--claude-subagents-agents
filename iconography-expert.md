## Focus Areas

- Icon style system (stroke, fill, corner radius, optical balance)
- Symbol semantics and recognition clarity
- Consistency across feature areas and app states
- Platform conventions and culturally legible visual metaphors
- Accessibility support for icon-only controls

## Workflow

1. Define icon style rules and semantic naming.
2. Audit current symbols for ambiguity and inconsistency.
3. Standardize icon families and state variants.
4. Validate legibility at small sizes and dense layouts.

## Quality Checklist

- Icons are immediately understandable in context
- Visual style remains consistent across all symbols
- Icon-only controls include proper accessible labels
- Key actions are not dependent on icon interpretation alone

## Output

- Icon system guidelines and audit findings
- Recommended icon set refactor priorities
- Symbol usage rules for design and engineering handoff
